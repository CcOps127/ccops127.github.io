安装环境准备

  • minikube

Minikube 是一个工具，可以快速运行一个单节点的 kubernetes，尝试 kubernetes或日常开发的用户适用，不适用于生产环境。

官方地址: https://kubernetes.io/docs/setup/minikube/

  • kubeadm

kubeadm 也是一个工具，提供 kubeadm init 和 kubeadm join，用于快速部署 kubernetes 集群
官方地址: https://kubernetes.io/docs/reference/setup-tools/kubeadm/kubeadm/

  • 二进制包

从官方下载的发行版二进制包，手动部署 kubernetes 的每一个组件，组成kubernetes 集群。

生产环境中部署 kubernetes 集群，只有 kubeadm 和 二进制包两种方式可选，kubeadm 降低了部署门槛，但是屏蔽了很多细节，当遇到问题时很难排查。使用二进制包安装，虽然麻烦，但是可以学到很多工作原理，更有利于后期维护。

部署 flannel 网络

kubernetes 要求集群内各个节点能够通过 Pod 网络互联互通，flannel 使用 vxlan 技术为各个节点创建一个可以互通的 pod 网络。

本次安装涉及节点，ali-sh-d-k8s-70 172.16.31.70、ali-sh-d-k8s-71 172.16.31.71

flannel 最新版本可以查看 https://github.com/coreos/flannel/releases

签发证书

etcd 集群启用了双向 TLS 认证，所以需要为 flanneld 指定与 etcd 集群通信的 CA 证书和密钥。

创建 flannel 证书签发请求

向 etcd 集群写入集群 pod 网络信息

flannel 要适用 etcd 存储自身一个子网信息，要保证可以成功连接 etcd，写入预定义子网网段。

  • flanneld 当前版本 (v0.11.0) 不支持 etcd v3，故使用 etcd v2 API 写入配置 key 和网段数据；
  • 写入的 Pod 网段 CLUSTER_CIDR 必须是 /16 段地址，必须与 kube-controller-manager 的 –cluster-cidr 参数值一致；

配置 flannel

创建 flanned 的 systemd 文件

  • mk-docker-opt.sh 脚本将分配给 flanned 的 pod 子网信息写入 /run/flannel/docker 文件中，后续 docker 在启动时，会使用这个文件中的环境变量配置的 docker 网桥。
  • flanneld 使用系统缺省的路由所在接口与其他节点通信，对于有多个网络接口的节点，可以使用 -iface 参数指定通信接口，如 eth0
  • flanneld 运行时需要 root 权限
  • -ip-masq: flanneld 为访问 Pod 网络外的流量设置 SNAT 规则，同时将传递给 Docker 的变量 –ip-masq（/run/flannel/docker 文件中）设置为 false，这样 Docker 将不再创建 SNAT 规则；

Docker 的 –ip-masq 为 true 时，创建的 SNAT 规则比较“暴力”：将所有本节点 Pod 发起的、访问非 docker0 接口的请求做 SNAT，这样访问其他节点 Pod 的请求来源 IP 会被设置为 flannel.1 接口的 IP，导致目的 Pod 看不到真实的来源 Pod IP。 而 flanneld 创建的 SNAT 规则比较温和，只对访问非 Pod 网段的请求做 SNAT。

部署 node 节点

Master apiserver 启动TLS 认证后，Node 节点 kubelet 组件想要加入集群，必须适用 CA 签发的证书才能和 apiserver 通信。当 node 节点很多时，签发证书也是一个很繁琐的事情，因此有了 TLS Bootstrapping 机制，kubelet 会以一个低权限的用户自动向 apiserver 申请证书，kubelet 的证书有 apiserver 动态签发。

node 节点运行组件如下：

  • docker
  • flannel
  • kubelet
  • kube-proxy

kubectl 默认从 ~/.kube/config 文件读取 kube-apiserver 地址、证书、用户名等信息，如果没有配置，执行 kubectl 命令时可能会出错：

将kubelet-bootstrap用户绑定到系统集群角色

创建 kubeconfig 文件

kubeconfig 为 kubectl 的配置文件，包含访问 apiserver 的所有信息，如 apiserver 地址、CA 证书和自身使用的证书；

  • --certificate-authority：验证 kube-apiserver 证书的根证书；
  • --client-certificate、--client-key：刚生成的 k8s-admin 证书和私钥，连接 kube-apiserver 时使用；
  • --embed-certs=true：将 ca.pem 和 k8s-admin.pem 证书内容嵌入到生成的 kubectl.kubeconfig 文件中(不加时，写入的是证书文件路径)；

部署 kubelet 组件

kubelet 运行在每个 node 节点，接受 kube-apiserver 发送的请求，管理 Pod 容器，执行交互命令，如 exec、run、logs等。
kubelet 启动时自动向 kube-apiserver 注册节点信息，内置的 cadvisor 统计和监控节点的资源使用情况，为了安全，只开启接受 https 请求的安全端口，并对请求进行认证和授权，拒绝未授权的请求。
