etcd 提供了多钟方式来构建高可用集群，在「基于静态发现方式部署Etcd集群」中介绍了通过静态方式部署集群。

如果你只知道需要搭建一个集群，但是不能事先知道集群各个节点的 ip ，从而无法使用 --initial-cluster 参数。这个时候，就需要使用 Discovery 模式来构建 etcd 集群。discovery 模式分为两种 etcd Discovery 和 DNS Discovery。

本文介绍通过 etcd Discovery 方式构建 etcd 集群。这个启动方式需要依赖应为一个现有的 etcd 集群，在集群中创建一个工作目录，并创建一个 _config的子目录。并且在该子目录中增加一个 size节点，指定集群中的节点数量。在这种情况下，将该目录在 etcd 中的 URL 作为节点的启动参数，即可完成集群启动。

值得注意的是：如果实际启动的 etcd 节点个数大于 discovery token 创建时指定的 size，多余的节点会自动变为 proxy 节点。

etcd proxy模式简介：
作为反向代理把客户的请求转发给可用的etcd集群，新节点加入集群如果核心节点数已满足要求，则自动转化为proxy模式，此项不会在节点不足时逆向转化为实际节点。
