基于静态发现方式部署Etcd集群

发表于 2019-02-22 | 更新于 2019-05-20 | 分类于 技术之道 | 阅读次数：
本文字数： | 阅读时长 ≈

基于「Etcd 入门使用」文中介绍的 etcd 基本知识点和安装，部署一个 etcd 集群。

etcd 构建高可用集群有三种形式：

  • 静态发现：预先已知 etcd 集群中有那几个节点，在启动时通过 --initial-cluster 参数直接指定好 etcd 的各个节点地址。
  • 动态发现：通过已有的 etcd 集群作为数据交互点，通过已有的 etcd 集群进行服务发现，扩展现有集群。例如官方提供的discovery.etcd.io
  • DNS 动态发现，通过 DNS 查询方式获取其他节点地址信息

参数选项解释

针对上面使用到的几个参数选项如下：

  • ETCD_NAME: etcd 集群的节点名
  • ETCD_DATA_DIR: etcd 集群的数据存储目录
  • ETCD_LISTEN_PEER_URLS: 该节点与其他节点通信时监听的地址列表，多个地址使用逗号分隔 [http|https]://IP:PORT。
  • ETCD_LISTEN_CLIENT_URLS: 该节点与客户端通信时监听的地址列表
  • ETCD_INITIAL_ADVERTISE_PEER_URLS: 该成员节点在整个集群中的通信地址列表，这个地址用来传输集群数据的地址。因此这个地址必须是可以连接集群中所有的成员的。
  • ETCD_INITIAL_CLUSTER: 配置集群内部所有成员地址，语法ETCD_NAME=ETCD_INITIAL_ADVERTISE_PEER_URLS，多个用逗号分隔。
  • ETCD_INITIAL_CLUSTER_STATE: 初始化集群状态，new 表示新建
  • ETCD_INITIAL_CLUSTER_TOKEN: 初始化集群 token
  • ETCD_ADVERTISE_CLIENT_URLS: 广播给集群中其他成员自己的客户端地址列表
  • ETCD_SNAPSHOT_COUNTER：多少次的事务提交将触发一次快照
  • ETCD_HEARTBEAT_INTERVAL：etcd 集群节点之间心跳传输的间隔，单位毫秒
  • ETCD_ELECTION_TIMEOUT：该节点参与选举的最大超时时间，单位毫秒

注意：所有 ETCD_MY_FLAG 的配置参数也可以通过命令行参数进行设置，但是命令行指定的参数优先级更高，同时存在时会覆盖环境变量对应的值。
