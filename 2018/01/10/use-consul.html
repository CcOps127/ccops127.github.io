安装 Consul

Consul 必须安装在 Consul 集群中的每个节点。为了安装方便，consul 为所有支持的 平台和架构分发一个二进制包，下载地址：https://releases.hashicorp.com/consul/。

安装 Consul

  • 安装

安装 consul，要找到适合操作系统的包，并下载它。consul 被打包挣了一个 zip 包。 下载之后，解压压缩包。将 consul 的二进制执行程序放到 /usr/local/bin/

  • 验证 安装之后的验证，可以通过打开新的终端会话来检查 consul 是否可用。通过执行 consul命令 可以看到类似下面的输出

运行 Agent

Consul 安装之后，agent 必须运行，agnet 可以运行在 server 和 client 模式下。 每个数据中心至少要由一个server，不过一个集群推荐 3-5 个 server。单一server的部署， 使出现故障后数据的丢失难以避免。 所有 agent 运行在 client 模式，一个client是一个非常轻量级的进程，用来注册服务， 运行健康检查，以及转发查询到server。

启动 Agnet

consul agnet 启动后会输出一些日志，从日志看来，agent 正在运行在 server 模式， 在集群中为 leader身份，另外，本地成员标记为集群中的一个健康成员

集群成员

如果在另外一个终端中运行 consul menbers，可以看到当前consul 集群中的成员，目前只有 它自己。

在输出中，可以看到自己的节点、运行的地址、健康状态、集群角色，以及一些版本信息，此外元 数据可以通过提供的detailed来查看。

members命令输出是基于留言协议，是最终一致的。因此，在某个时间点，本地的 agent的世界版图 可能不能严格的匹配server状态。要获取一个强一致性的世界版图，使用HTTP API，它会转发请求 到consul server

除了通过HTTP API，dns接口亦可以来查询节点。需要确保 DNS 查询指向consul agent 的dns服务器，默认运行在 8600 端口。

停止 agent

通过 Ctrl+c 来终止 agent,在终止后，consul 会通知其他集群中的成员这个节点离开了， 如果强制杀死进程，集群中的成员检测到节点失败，他的服务和检查将从登记目录中移除，当一个成员失败， 它的健康状态会简单的标记为critical，但是它不会被从登记目录中移除。Consul将会自动重新连接失败的节点， 允许它从一定的网络状态下恢复回来，而离开的节点将不会再进行联系。此外，如果Agent作为Server运行， 优雅地离开是很重要的，可以避免因影响一致性协议导致的可能的可用性中断。

服务管理

注册服务

注册服务可以通过提供的一个服务定义的方式，还可以提供过调用HTTP API的方式。 服务定义是注册服务的最常见方式，可以在 anget配置中来构建它。

  • 定义服务

首先，为consul配置一个目录，consul会从这个目录中加载配置文件。

定义一个 web 的服务，运行在 80端口上，标签为web-test，作为一个附加方式查询服务。

  • 启动 agent

当日志中出现 agent: Synced service "web" 说明 agent 已经从配置文件中加载了服务定义信息， 并且已经注册到服务条目。如果想要注册多个服务，可以创建多个服务定义文件。

查询服务

agent 启动并且服务被同步，就可以使用DNS或者HTTP API的方式来查询服务

  • DNS 接口
  • HTTP 接口

还可以使用 HTTP API来查询服务

接口catelog返回指定服务的所有宿主节点，通常只是用来获取通过监控检查的健康实例

更新服务

可以通过更改配置文件，然后发送一个SIGHUP信号给agent来更新服务。这样可以在没有停机或 服务查询不可用的情况下更新服务。此外。HTTP API还可以动态的添加、移除、修改服务。

Consul集群管理

当一个 consul agent 启动后，它没有其他节点的任何信息。是一个孤立的集群，如果想要知道 其他集群成员，需要加入一个集群。在加入集群后，可以通过流言协议和成员之间通信，并发现集群 中的其他成员。consul agnet 可以加入任何一个其他的agent，不止是server模式的 agent

启动 Agent

  • -node 集群中的每个节点都需要一个唯一的名字，默认会使用consul的主机名，可以通过使用-node选 项来自定义。
  • -bind 指定一个绑定地址，这个是 consul的监听地址
  • -server 指定集群中的唯一server
  • -bootstrap-expect 告诉consul server要加入集群的数量
  • -config-dir 指定服务定义配置文件

加入 cluster

  • 手动加入集群

可以在agent的日志中看到，节点二成功记入到集群中。

  • 启动时自动加入集群

理想情况下，当一个新的节点在数据中被创建，应该可以自动加入consul集群中，而不需要人工 干预，可以通过使用 -join参数或者start_join设置已知的consul agent地址

查询节点

就像查询服务，consul由一个API来查询节点。

除了用于服务查找，节点查找功能对于系统管理任务也很有用。例如，知道该节点的地址进行 SSH 登入将像标记该即诶但作为consul集群一部分并查询它一样简单。

离开集群

要离开集群，可以优雅的退出agnet,也可以强制杀死 agnet。优雅的离开状态是left,否则 其他节点将会检测到它并标记failed状态

健康检查

健康检查对于服务发现是一个关键的组件，它可以防止使用不健康的服务

定义健康检查

和服务类似，一个检查也可以通过定义来实现，还可以通过HTTP API来定义，

第一个命令是添加一个名为”ping” 的主机级别检查,这个检查每个 30s 运行一次，调用 ping -c1 baidu.com。 对于基于脚本的健康检查，检查使用和启动consul进程相同的用户进行运行。如果这个命令返回一个 非零退出码，这个节点将被标记为不健康。对于基于脚本的检查，这是约定。

Consul Web 用户界面

Consul 自带一个漂亮的、多种功能的 web 用户界面。用户界面可以用来浏览所有的服务和节点， 浏览所有的健康检查和他们最新的状态，以及读写键值数据。用户界面还自动支持多数据中心。 运 要设置自宿主用户界面，启动Consul Agent时使用-ui参数。
